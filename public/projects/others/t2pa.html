Bibliotecas/JARs externos

Todas as bibliotecas utilizadas se encontram no diretório 'src/', na raiz do projeto.

opencsv-2.2.jar

Simples parser de arquivos CSV utilizado para exportação do catálogo.

itextpdf-5.1.0.jar

Biblioteca utilizada para realizar a conversão do arquivo CSV para PDF.

commons-fileupload-1.2.jar

Bibliotea comum utilizada para importar as funções de de upload de arquivo padrões do Java

commons-io-2.4.jar

Biblioteca comum utilizada para importar funções de escrita em disco do Java

cos.jar

Biblioteca utilizada para obter parametros do form multpart provenientes do cliente e definir tamanho máximo de upload

Setup

Ambiente de desenvolvimento

O projeto foi desenvolvido utilizando as técnologias pedidas em sua especificação, na IDE Netbeans em um sistema Linux (Ubuntu14.04). O banco utilizado foi o Postgresql em modo autocommit ativado pelo driver. O servidor utilizado foi o Glassfish4.1 integrado ao Netbeans com Java EE 7 Web.

Configurações

Algumas classes devem ter seus parâmetros alterados manualmente.

A classe DadosBanco.java deve ser preenchida com os dados do banco.

A classe DadosArquivos.java deve ser preenchida com caminhos absolutos referentes ao servidor. O primeiro deles, 'caminhoArquivos', se refere a um diretório qualquer em seu sistema, a sua escolha, sob o qual o banco de dados tenha permissão de escrita. Essa pasta seria o local onde qualquer processamento externo sobre os arquivos tomaria lugar. O segundo, 'caminhoArquivosWeb', se refere à pasta na qual os arquivos enviados pelos clientes serão armazenados e distribuídos.

Banco de Dados

Modelo

O modelo utilizado é exatamente o mesmo que o inicialmente fornecido Pelo professor.

Atenções

As classes que lidam com o banco de dados forma idealizadas de maneira completamente idependentes umas das outras e com nomeclatura de fácil leitura, sempre (e ecxepcionalmente) em potuguês.

A classe Buscador.java foi feita de forma a gerar querys de busca dinâmicas, o que contribui para seu grande tamanho e difícil legibilidade

A classe ExportarCatalogo.java é responsável por gerar o arquivo CSV e convertê-lo para PDF.

Servidor

Modelo

As classes do modelo incluem as que interagem com o banco de dados, as que armazenam informações locais e algumas outras de tratamento de repasse de dados.

Controller

Principal Servlet do projeto. Recebe todas as requisições AJAX e as trata com um identificador "tipo" enviado pelo cliente como um atributo Json, utilizando a classe JsonObject. Envia respostas na forma de arrays json, utilizando a classe JsonArray.

UploadFile

Servlet dedicado ao upload de arquivos. Recebe requisições multipart e as trata de acordo, sem responder diretamente ao cliente o sucesso da transmissção (o qual é indicado ao cliente por meio do form jquery).

Críticas e Melhorias

Críticas

Por se tratar de uma aplicação desenvolvida como parte de um curso universitário, sem um forte compromisso com a usabilidade final do usuário e, também, considerando o tempo limitado, alguns aspectos do projeto foram, até certo ponto, negligenciados.

O primeiro e mais evidente deles é o sistema de busca, que considera apenas as palavras exatas presentes no banco de dados. Normalização de caracteres e busca por palavras seriam métodos fáceis de minimizar o impacto dessa decisão de design, contudo, julguei não valerem a pena pelo esforço necessário, na modificação desde o banco até a página acessada pelo cliente.

Outro ponto que atrapalha a usabilidade do cliente é o fato de que algumas das funções javascript não promoverem a atualização instantânea da página após a realização de uma certa ação. é uma questão de timing que ainda não acertei, embora funcionem na maior parte das vezes.

Por fim, restam questões como layout da página, flow visual e embelezamento a serem mais discutidas e melhoradas, mas que levariam muito tempo até serem acertadas.

Essa é a versão beta v0.2 e, como sempre é dito na aula, você só acerta (quase) tudo na quinta ou sexta vez que está refazendo o projeto.

Futuro

Alguns pontos valem a pena ser levantados. O primeiro se trata do upload de arquivos. No modelo atual, os arquivos são recebidos e depois copiados para o diretório de distribuição. Ainda nesse ponto, era de meu interesse permitir a encriptação do arquivo com uma senha fornecida pelo usuário, mas não tive tempo de investigar essa ideia a fundo.

Fora isso, seria interessante aprimorar as funcionalidades extras que adicionei ao projeto, como especificar o tipo de exportação do catálogo em pdf e aprimorar o sistema de comentários por meio da implementação efetiva de um banco de cliente/usuários.
